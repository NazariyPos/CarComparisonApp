# Стратегія логування (Serilog)

## 1) Типи помилок у проєкті

Для `CarComparisonApi` актуальні такі групи помилок:

1. **Валідаційні помилки запиту**
   - некоректні параметри пошуку;
   - некоректний рейтинг `review`;
   - невідповідний формат логіна/пароля.

2. **Помилки автентифікації/авторизації**
   - неправильні креденшали;
   - відсутній/прострочений JWT;
   - відсутні права доступу.

3. **Помилки доступу до даних**
   - відсутній `Data/*.json`;
   - помилка читання/запису JSON;
   - пошкоджений JSON формат.

4. **Інфраструктурні та runtime помилки**
   - неочікувані винятки під час запуску/зупинки;
   - помилки middleware/пайплайна;
   - залежності недоступні або некоректно налаштовані.

## 2) Рівні логування

Використані базові рівні:

- `DEBUG` — технічні деталі (збереження файлів, допоміжні операції).
- `INFO` — нормальні бізнес-події (успішний логін/реєстрація, створення/оновлення/видалення даних).
- `WARNING` — підозрілі або очікувано-проблемні сценарії (невдалий логін, спроба оновити неіснуючий запис).
- `ERROR` — помилки, що зривають поточну операцію (помилка читання/запису даних, винятки сервісів).
- `CRITICAL` (`Fatal` у Serilog) — критичне падіння застосунку або неможливість продовжити роботу.

## 3) Інтеграція Serilog у проєкт

Реалізовано:

1. Підключено пакети:
   - `Serilog.AspNetCore`
   - `Serilog.Settings.Configuration`
   - `Serilog.Sinks.Console`
   - `Serilog.Sinks.File`

2. Налаштовано базові рівні та overrides у `appsettings.json`:
   - `Default=Information`
   - `CarComparisonApi=Debug`
   - `Microsoft/Microsoft.AspNetCore/System=Warning`

3. Налаштовано формат логів:
   - `Timestamp`
   - `Level`
   - `SourceContext` (модуль)
   - `Message`
   - `Exception`

4. Увімкнено вивід у:
   - консоль
   - файл `logs/carcomparison-.log` (щоденна ротація)

## 4) Де додано базове логування

1. **Запуск та зупинка програми** (`Program.cs`):
   - старт хоста;
   - події `ApplicationStarted`, `ApplicationStopping`, `ApplicationStopped`;
   - `Fatal` при неочікуваному завершенні.

2. **Критичні операції з даними**:
   - `JsonUserService`: завантаження/збереження/створення/оновлення/видалення користувачів;
   - `ReviewService`: завантаження/збереження/CRUD відгуків;
   - `AuthService`: реєстрація, логін, доступ до користувачів.

3. **Помилки та винятки**:
   - логування помилок читання/запису файлів (`Error`);
   - логування винятків у сервісах (`Error`);
   - фатальні помилки життєвого циклу (`Fatal`).
